Se a foto está cortando, é porque o comando background-size: cover tenta preencher a tela inteira e acaba cortando as bordas da imagem para não deixar espaços vazios. Para que a foto apareça inteira (sem cortes), precisamos mudar para contain. Fiz também um ajuste no fundo para não ficar com faixas brancas nas laterais, usando um tom que combina com o tema. Substitua o bloco de estilo do body e da .hero-inicial por este: CSS body { /* Ajustado para 'contain' para a foto aparecer inteira sem cortes */ background-image: url('./assets/fotos/foto3.jpg'); background-size: contain; background-position: center; background-attachment: fixed; background-repeat: no-repeat; background-color: #f9f9f9; /* Cor de fundo caso a foto não ocupe a tela toda */ color: #004d40; font-family: 'Lato', sans-serif; overflow-x: hidden; display: flex; flex-direction: column; align-items: center; min-height: 100vh; } .hero-inicial { height: 100vh; width: 100%; display: flex; flex-direction: column; justify-content: flex-end; /* Move o texto e botão para baixo para não tampar o rosto na foto */ align-items: center; text-align: center; background: transparent; padding: 50px 20px; /* Espaçamento inferior */ position: fixed; z-index: 2000; top: 0; left: 0; } O que mudou: background-size: contain: Agora o navegador vai reduzir a foto até que ela caiba inteira na tela, sem cortar nada. justify-content: flex-end: Mudei a posição do texto e do botão para o final da tela (parte de baixo). Como a foto agora aparece inteira, geralmente o rosto das pessoas está no centro/cima, então colocar os elementos embaixo evita que eles fiquem em cima da imagem. background-color: Adicionei um cinza bem clarinho para que, se a foto for vertical e sobrar espaço nas laterais, não fique um visual "quebrado". Teste assim e veja se agora a foto aparece completa!
